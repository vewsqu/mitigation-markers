#  Mitigation Markers Detector

**Корпусный инструментарий для автоматического выделения маркеров митигации в русскоязычных новостных статьях.**

Проект представляет собой NLP-пайплайн (rule-based подход), предназначенный для автоматического обнаружения, классификации и количественного анализа языковых средств митигации (хеджей) в текстах СМИ. Разработан в рамках исследования медиадискурса на базе Санкт-Петербургского политехнического университета Петра Великого (СПбПУ).

##  Возможности проекта

*   **Веб-скрейпинг:** Автоматизированный сбор новостных текстов с сайтов РБК и «Коммерсантъ» (с помощью `Scrapy`).
*   **Умная предобработка текста:** Очистка текстов от пунктуации с помощью регулярных выражений (с сохранением внутрисловных дефисов для корректной лемматизации).
*   **Морфологическая нормализация:** Токенизация и лемматизация русскоязычного текста с использованием `pymorphy3`.
*   **Многоклассовая классификация:** Распознавание 4 функциональных типов маркеров митигации:
    *   `source` (ссылки на источник)
    *   `epistemic` (эпистемические хеджи)
    *   `conditional` (условные конструкции)
    *   `deagent` (деагентивация)
*   **Статистический анализ:** Подсчет количества маркеров и нормализованной плотности (density) на 1000 слов для корректного сравнения текстов разного объема.
*   **Визуализация:** Автоматическое построение графиков распределения (`matplotlib`).

##  Технологический стек

*   **Язык:** Python 3.9+
*   **NLP и Лингвистика:** `pymorphy3`, `re` (регулярные выражения)
*   **Скрейпинг:** `Scrapy`
*   **Анализ данных и визуализация:** `pandas`, `matplotlib`

##  Структура данных

В процессе работы алгоритм генерирует два выходных файла:
1.  `markers.csv` — лог всех найденных маркеров с привязкой к ID текста и функциональному классу (в форме инфинитивов).
2.  `texts_stats.csv` — статистика по проанализированному корпусу (количество слов, количество найденных маркеров, плотность маркеров на 1000 слов).

##  Как работает алгоритм (NLP Pipeline)

1. **Очистка (Regex):** Из входного текста удаляются все знаки препинания, кроме дефисов `re.sub(r'[^\w\s-]', '', text)`. Это решает проблему «прилипания» пунктуации к токенам.
2. **Лемматизация:** Каждое слово приводится к начальной форме с помощью `pymorphy3` (например, *«сообщила»* $\rightarrow$ *«сообщить»*).
3. **Поиск по паттернам:** Скользящее окно сопоставляет n-граммы текста с заранее заданным «золотым стандартом» (словарем маркеров).
4. **Агрегация:** Формируются DataFrame-таблицы и рассчитывается дискурсивная плотность текстов.

##  Известные ограничения

Текущая реализация основана на словарном (rule-based) методе. В связи с этим алгоритм:
* Не выполняет синтаксический анализ, что может приводить к прагматической омонимии (False Positives).
* Зависит от видовых пар глаголов в словаре (необходимо вручную задавать леммы и совершенного, и несовершенного вида).

В будущих релизах планируется переход к векторным представлениям слов (Word2Vec/Transformer-модели) для учета семантического контекста.
